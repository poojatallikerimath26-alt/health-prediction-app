# Health Prediction Application

A simple CRUD web app that stores patient blood-test records and uses a
trained machine learning model to automatically predict possible health
risks, filling in the **Remarks** field.

Built for the Gokul Infocare **Junior AI/ML Developer – Task 1** technical
assessment.

## Tech Stack

| Layer        | Choice                              |
|--------------|--------------------------------------|
| Backend      | Python 3 + Django                    |
| Frontend     | Django Templates + Bootstrap 5       |
| Database     | MySQL (SQLite supported for quick local testing) |
| AI/ML        | scikit-learn Logistic Regression (3 trained models) |

**Why this stack:** Django, MySQL, and basic frontend templating are
already part of my skill set, so this stack let me focus the available
time on getting the CRUD flow and the AI/ML integration fully correct
rather than learning a new framework under deadline.

## Why a trained model instead of a public API

Most free "health prediction" APIs either require a paid key, have
unreliable uptime, or don't really exist for this exact use case. The
task description explicitly allows a **custom ML model** as an
alternative, so I trained three small Logistic Regression classifiers
(diabetes risk, anaemia risk, high cholesterol risk) on a synthetic
dataset built from standard clinical reference ranges:

- Glucose ≥ 126 mg/dL → diabetes risk
- Haemoglobin < 12 g/dL → anaemia risk
- Cholesterol ≥ 240 mg/dL → high cholesterol risk

A small amount of random label noise was added during training so the
models learn an actual decision boundary instead of just memorising the
threshold — see `patients/ml/train_model.py` for the full logic. This
keeps the project fully self-contained (no external API keys, no
internet dependency, nothing to break during a demo) while still being a
genuine ML pipeline rather than hardcoded if/else statements.

`patients/ml/predictor.py` loads the saved models and turns their
predictions into the readable sentence that gets saved into the Remarks
field whenever a record is created or updated.

## Features

- Full CRUD: create, view, edit, and delete patient records
- Auto-generated Remarks field powered by the ML models above
- Validation: date of birth can't be in the future, email must be valid,
  glucose/haemoglobin/cholesterol must be positive numbers
- Clean Bootstrap UI
- Django admin panel for an alternate view of the data

## Setup & Running Locally

1. **Clone and create a virtual environment**
   ```bash
   git clone <your-repo-url>
   cd health_prediction_app
   python -m venv venv
   source venv/bin/activate      # Windows: venv\Scripts\activate
   ```

2. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure environment**
   ```bash
   cp .env.example .env
   ```
   By default `DB_ENGINE=sqlite`, so no database setup is required to try
   it out. To use MySQL instead, install/start MySQL, create a database,
   and set `DB_ENGINE=mysql` plus the `DB_*` values in `.env`.

4. **Train the ML models** (only needs to be run once — the trained
   `.pkl` files are not committed to GitHub on purpose, so this step is
   required after cloning)
   ```bash
   python patients/ml/train_model.py
   ```

5. **Run migrations and start the server**
   ```bash
   python manage.py migrate
   python manage.py runserver
   ```

6. Open `http://127.0.0.1:8000/` in your browser.

## Notes

- No passwords, API keys, or secrets are committed — `.env` is
  git-ignored, and `.env.example` only contains placeholders.
- This app intentionally does not use any real medical API and does not
  provide real medical diagnoses; the "AI/ML" predictions are for
  demonstration purposes only, based on standard public clinical
  reference ranges.
